# Cursor Configuration for Naylence Website

This directory contains Cursor-specific configuration and documentation for the Naylence website project.

## Files Overview

### `.cursorrules`
Main configuration file that Cursor AI reads automatically. Contains:
- Project overview and tech stack
- Design system and branding guidelines
- Code style and conventions
- Component patterns and best practices
- Common workflows and troubleshooting

### `prompts.md`
Quick reference prompts for common tasks:
- Adding new features
- Styling components
- Creating responsive layouts
- Component-specific prompts
- Code quality prompts

### `commands.md`
Common commands and shortcuts:
- Development commands
- Git workflow
- Vercel deployment
- Code quality tools
- Debugging commands
- Browser testing

### `instructions.md`
Detailed development instructions:
- Getting started guide
- Development workflow
- Component development
- Content management
- Testing procedures
- Deployment guide

## Quick Start

1. **Read `.cursorrules`** - Understand the project structure and conventions
2. **Reference `prompts.md`** - Use prompts when asking Cursor AI for help
3. **Check `commands.md`** - Find the right command for your task
4. **Follow `instructions.md`** - Step-by-step guides for common tasks

## Using with Cursor AI

When working on this project, Cursor AI will automatically:
- Read `.cursorrules` for context
- Follow the design system and patterns
- Suggest code that matches the project style
- Provide relevant prompts and commands

You can also explicitly reference these files:
- "Follow the patterns in `.cursorrules`"
- "Use prompts from `prompts.md`"
- "Check `commands.md` for the right command"

## Updating Documentation

When making significant changes to the project:
1. Update `.cursorrules` with new patterns or conventions
2. Add new prompts to `prompts.md` if needed
3. Document new commands in `commands.md`
4. Update `instructions.md` with new workflows
